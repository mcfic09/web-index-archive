# web-index-archive

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件的形式存储，可通过 GitHub Pages 等静态托管服务直接访问。

仓库本身不针对任何特定域名或网站，仅作为 HTML 内容的集中存放与版本管理场所。

## 目录结构

```
├── index.html          # 仓库根目录的入口页面（如有）
├── pages/              # 存放主要 HTML 页面文件的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 静态资源文件夹（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

各目录及文件命名以简洁、清晰为原则，便于维护和引用。

## 页面归档说明

- 归档的 HTML 页面均为独立文件，不依赖外部数据库或后端服务。
- 页面间的链接采用相对路径，确保在本地或静态托管环境中均可正常导航。
- 所有页面内容遵循通用 HTML 规范，未使用特殊框架或构建工具，保持通用可读性。

## 使用方式

1. **本地预览**：将仓库克隆到本地，直接用浏览器打开 HTML 文件即可查看。
2. **在线部署**：启用仓库的 GitHub Pages 功能，选择 `main` 分支（或 `docs/` 目录）作为源，即可通过 `https://<用户名>.github.io/web-index-archive/` 访问。
3. **自定义域名**：如有需要，可在仓库设置中绑定自定义域名（但本仓库不预设或推荐任何域名）。
